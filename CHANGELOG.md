## v0.8.2（2026-06-25）

### 优化

- **加载进度条优化**：alldata.js（~5MB）下载期间进度条通过 CSS 动画自动从 10% 推进至 30%，避免卡在 10% 超过 1 秒无响应的体验。JS 接管后平滑过渡到真实进度。
- **百分比数字动态显示**：初始改为「载入中…」，JS 启动后立刻显示当前真实百分比，不再硬编码 10%。
- **提前预加载数据文件**：`<link rel="preload">` 让浏览器在 HTML 解析阶段即开始下载 alldata.js，缩短实际等待时间。

## v0.8.1（2026-06-25）

### 新增

### 工作区整理

- 将与网页无关的脚本、原始数据和旧的 changelog 归档至 `data/` 目录，`.gitignore` 排除 `data/` 目录，不再纳入版本管理。
- 移除对 `985&211.json`、`changelog/index.html` 的 git 跟踪。

## v0.6.1（2026-06-23）

### 新增算法模式

- **仅2024**：仅使用2024年录取数据作为参考依据，不涉及2025年及2026年预估数据
- **仅2025**：仅使用2025年录取数据作为参考依据，不涉及2024年及2026年预估数据

### 修复

- **非默认算法不再推荐"新"专业**：按最低/平均/最高/仅2024/仅2025 模式下，无历史数据的2026年新增院校专业组（"新"标签）不会出现在推荐结果中
- **非默认算法不计入2026预估分**：计算冲稳保时不再将2026年预估分数纳入考虑，确保严格仅使用选中的历史年份数据

## v0.6.0（2026-06-23）

### 志愿推荐算法切换（Picker 弹窗选择）

- 推荐算法从标签切换改为 Picker 弹窗样式（与选科要求一致）
- 弹窗内每条显示标题 + 详细说明，帮助考生理解各算法差异
- 4 种算法模式：
  - **默认（可推荐新专业）**：优先 2026 预估排名，回退 2024-2025 平均排名，可推荐新增专业
  - **按最低（最激进）**：仅用 2024-2025 最低排名，不推荐新增专业
  - **按平均**：仅用 2024-2025 平均排名，不推荐新增专业
  - **按最高（最保守）**：仅用 2024-2025 最高排名，不推荐新增专业
- 排序（各档内）同步根据算法选取参考分数
- 服从调剂按钮移至算法选择器右侧同排，标签简化为"服从调剂"

### 修复

- **修正算法标签**：最低分模式(`min`)参考的录取门槛最低，判定更多结果为"冲"，实际为激进策略；最高分模式(`max`)参考的录取门槛最高，判定更多结果为"保"，实际为保守策略。已修正标签和描述文案。
